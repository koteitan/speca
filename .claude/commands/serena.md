[## Usage Patterns Guide

### 1. Basic Usage
```bash
# Simple problem solving
/serena "Fix login bug"

# Rapid feature implementation
/serena "Add search filters" -q

# Code optimization
/serena "Improve load time" -c
```

### 2. Advanced Usage
```bash
# Complex system design (with research)
/serena "Design a microservices architecture" -d -r -v

# Full feature development (with TODO creation)
/serena "Implement a user dashboard with charts" -s -t -c

# In-depth analysis (with documentation)
/serena "Migrate to a new framework" -d -r -v --focus=frontend
```

## Automatic Selection by Problem Type

Serena automatically selects the optimal reasoning pattern based on **keywords**:

### Debug Pattern (5-8 reasoning steps)
**Keywords**: error, bug, issue, broken, failing
```bash
/serena "An API error is occurring"
# → Symptom analysis → Environment check → Hypothesis → Verification → Solution → Implementation → Test strategy → Preventive measures
```

### Design Pattern (8-12 reasoning steps)
**Keywords**: architecture, system, structure, plan
```bash
/serena "Design a user authentication system"
# → Requirements → Constraints → Option generation → Evaluation → Tech selection → Design decision → Implementation phase → Risk mitigation
```

### Implementation Pattern (6-10 reasoning steps)
**Keywords**: build, create, add, feature
```bash
/serena "Add real-time chat feature"
# → Spec confirmation → Tech selection → Design → Dependencies → Implementation order → Test strategy → Error handling
```

### Optimization Pattern (4-7 reasoning steps)
**Keywords**: performance, slow, improve, refactor
```bash
/serena "Optimize database queries"
# → Current state analysis → Bottleneck identification → Improvement opportunities → Solution options → Prioritization → Implementation impact assessment
```
