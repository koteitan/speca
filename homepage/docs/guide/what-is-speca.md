# これは何ですか？

SPECA は、書かれたコードが「要件どおりに動いているか」を自動で確認するツールです。

## 従来のコード診断ツールとの違い

静的解析やリンター（eslint など）は「コードの書き方のルール」をチェックします。例えば「変数の型は正しいか」「使っていない変数がないか」といった具合です。しかし、これらは「仕様で定められた振る舞い」までは検証しません。

SPECA は逆方向に考えます。仕様書を読んで「ここでこういう処理が必要」と判定し、実装されているコードとを照らし合わせて「ズレ」を見つけます。

## 仕様を基点とする発想

例えば、あるシステムの仕様に「ユーザー認証の後にのみ、機密データへのアクセスを許可する」と書かれていたとします。SPECA は以下の手順で確認します。

1. 仕様を読んで「これは重要なセキュリティ要件だ」と認識
2. コードのどこがこの要件を実装しているか探す
3. すべての経路で「認証 → データアクセス」の順が守られているか証明してみる
4. 証明できない部分（穴）を見つけたら、それは脆弱性の候補

## 想定ユーザー

- **セキュリティ監査を担当している**: 手作業で仕様書とコードを比較する手間を減らしたい
- **バグバウンティに参加している**: 大規模コードベースから実装の穴を素早く見つけたい
- **自分たちの開発品質を確かめたい**: 要件どおりに実装されているか検証したい
