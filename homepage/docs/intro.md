# SPECA とは

SPECA (Specification-to-Property Agentic Auditing) は、自然言語の仕様書から形式的なセキュリティプロパティを導出し、実装がそれらを満たすかどうかを証明ベースの監査で検証する自動セキュリティ監査フレームワークです。

## 問題

既存のコード駆動型セキュリティツールは、既知のバグパターンやデータフロー異常を検出します。しかし仕様で規定されたシステムでは、脆弱性が「コードの書き方」ではなく「仕様が要求すること」に起因します。これらのツールは仕様レベルの不変式を表現する語彙を持たず、検出できません。

## アプローチ

1. **仕様から型付きプロパティを導出**: 仕様書をプログラムグラフに分解し、STRIDE + CWE Top 25 脅威モデルを適用
2. **プルーフ試行ベースの監査**: 各プロパティが実装で成立するか証明を試みる。証明の隙間 (proof gap) が候補検出
3. **構造的なFP削減**: 信頼境界・スコープチェック・デッドコード判定の3ゲート (recall-safe)

## 結果

- **Sherlock Ethereum Fusaka**: 全15個の対象脆弱性を検出。全366件の投稿を上回る4つの新規バグ発見 (開発者修正確認)
- **RepoAudit C/C++ベンチマーク**: 88.9% の精度を達成しつつ、確立された ground truth を超える12個の新規候補検出

## 対応言語

Go / Rust / Nim / TypeScript / C など仕様で規定されたシステム全般。
