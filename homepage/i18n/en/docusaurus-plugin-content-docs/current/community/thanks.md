# Thanks

SPECA stands on top of public datasets, benchmarks, and libraries from outside the project. This page acknowledges **the resources that sit outside SPECA itself**.

We do not list paper authors or repository contributors here. For paper links see the [References](../references/paper-fusaka), and for development history see [GitHub](https://github.com/NyxFoundation/speca).

## Benchmark and dataset providers

- **Sherlock** — Data from the Ethereum Fusaka audit contest
- **Code4rena** — Audit contest archives
- **CodeHawks** — Public audit result data
- **RepoAudit** — C/C++ project benchmark

Without these public resources, SPECA's evaluation and property extraction would not have come this far.

## Tools and libraries we depend on

- **Anthropic Claude Code** — Inference engine
- **Tree-sitter MCP** — Code location resolution in Phase 02c
- **uv** (Astral) — Python orchestrator runtime
- **Pydantic** — Inter-phase data contract validation
- **Docusaurus** — This site

## How to add entries

If an external resource is missing, please open a pull request that directly edits this file (`homepage/docs/community/thanks.md`) against [NyxFoundation/speca](https://github.com/NyxFoundation/speca).
